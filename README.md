# OLX Real Estate Scraper (Level 2 Technical Task)

Цей проект є технічним завданням (Level 2) для автоматизованого збору даних з каталогу нерухомості **OLX.ua** (Київ). Скрипт збирає інформацію про квартири, структурує її та автоматично завантажує у **Google Sheets**.

##  Основні можливості

* **Асинхронність:** Використання `playwright` разом із `asyncio.gather` та `Semaphore` дозволяє парсити кілька оголошень одночасно.
* **Гібридний підхід:** Поєднання `Playwright` (навігація) та `BeautifulSoup` (парсинг) забезпечує максимальну швидкість.
* **Оптимізація трафіку:** Реалізовано блокування картинок, стилів та шрифтів для економії ресурсів.
* **Автоматизація Google Sheets:** Інтеграція з `gspread` для запису даних.
* **Очищення даних:** Автоматична обробка цін та площ за допомогою регулярних виразів.

## Технологічний стек

* **Мова:** Python 3.13+
* **Бібліотеки:** Playwright, BeautifulSoup4
* **API:** Gspread (Google Sheets API), OAuth2
* **Асинхронність:** Asyncio, nest_asyncio

## Висновок

В ході виконання технічного завдання було розроблено асинхронний скрапер для збору даних з платформи **OLX.ua** (категорія "Квартири").

### 1. Які дані було спарсено?
Скрипт успішно збирає та структурує наступні атрибути оголошень:
* **Ціна:** Очищена від пробілів та службових слів (наприклад, "Договірна"), приведена до читого цифрового формату з валютою.
* **Поверх та Поверховість:** Точно розділені на дві окремі колонки завдяки аналізу текстових міток.
* **Район:** Витягнутий з останнього елемента навігаційного ланцюжка (breadcrumbs) для точної геоприв'язки.
* **Площа:** Тільки числове значення (м²), очищене від символів та готове для подальшої аналітики.
* **URL:** Пряме посилання на кожне оголошення.

### 2. Технічні рішення та оптимізація
* **Гібридна архітектура:** Використання **Playwright** для стабільної навігації та **BeautifulSoup** для миттєвої обробки HTML-коду. Це дозволило уникнути помилок "strict mode violation" та пришвидшити витягування даних.
* **Асинхронність:** Завдяки `asyncio.gather` оголошення обробляються паралельно. Використання `Semaphore(3)` забезпечує високу швидкість роботи без ризику блокування з боку захисних систем сайту.
* **Економія ресурсів:** Реалізовано перехоплення запитів браузера для блокування картинок, шрифтів та CSS-стилів. Це дозволило скоротити час завантаження сторінок та споживання оперативної пам'яті на **~70%**.
* **Надійність:** Додано обробку виняткових ситуацій (try-except) для кожного оголошення, що гарантує завершення роботи скрипта навіть при помилках на окремих сторінках.

**Результат:** Створено масштабований та швидкий інструмент, який повністю відповідає технічним вимогам **Level 2** та забезпечує структурований вивід даних у Google Sheets.
